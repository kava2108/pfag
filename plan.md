# PFAG — 実装計画（Implementation Plan）

本書は、`spec.md` に基づき PFAG を実装するためのフェーズ分解・マイルストーン・リスク・依存関係を示す。

---

## 1. 実装戦略

PFAG は以下のパイプライン構造で実装する：

1. PDF → 画像変換
2. 画像処理（下線・矩形検出）
3. 座標変換
4. AcroForm フィールド生成
5. PDF 組み立て
6. FastAPI による API 層
7. Agent Skills 連携

各フェーズは独立してテスト可能とする。

---

## 2. フェーズとマイルストーン

### **フェーズ1：プロジェクトセットアップ**
- リポジトリ構造の作成
- docs/・openapi/ の配置
- 依存ライブラリの導入（PyMuPDF, OpenCV, FastAPI, pdfrw）
- 開発環境の整備

**マイルストーン：** プロジェクト骨格が完成

---

### **フェーズ2：PDFレンダリングエンジン**
- PyMuPDF による PDF → 画像変換
- 300dpi レンダリング
- マルチページ対応
- 壊れたPDFの例外処理

**マイルストーン：** 全ページの画像生成が可能

---

### **フェーズ3：下線検出（テキストフィールド）**
- モルフォロジー演算の実装
- パラメータ（線幅・ギャップ）の導入
- 検出線をバウンディングボックス化
- フィールド配置ルールの実装

**マイルストーン：** テキストフィールド候補が安定検出

---

### **フェーズ4：チェックボックス検出**
- 輪郭抽出と階層解析
- アスペクト比・面積によるフィルタリング
- 正方形化処理
- サイズ制約の適用

**マイルストーン：** チェックボックス候補が安定検出

---

### **フェーズ5：座標変換**
- 画像座標 → PDF座標の変換ロジック実装
- アスペクト比差異の検証
- 単体テストの追加

**マイルストーン：** 正確な座標変換が可能

---

### **フェーズ6：AcroForm フィールド生成**
- `/Tx` `/Btn` フィールド生成
- 既存フィールドの保持
- 命名規則の実装
- Acrobat での動作確認

**マイルストーン：** 入力可能PDFの生成が可能

---

### **フェーズ7：API 層（FastAPI）**
- `POST /v1/formify` の実装
- multipart/form-data の処理
- options JSON の解析
- 警告ヘッダ `X-PFAG-Warning` の実装
- 200/422/500 のエラーハンドリング

**マイルストーン：** API が外部連携可能

---

### **フェーズ8：Agent Skills 連携**
- Skill スキーマの定義
- API 呼び出しラッパの実装
- 感度調整ロジック
- 警告メッセージのユーザー提示

**マイルストーン：** 会話型ワークフローが完成

---

## 3. リスクと対策

| リスク | 影響 | 対策 |
|-------|------|------|
| 細い線が検出されない | 中 | 感度調整・カーネル調整 |
| チェックボックス誤検出 | 中 | 輪郭フィルタ強化 |
| 大容量PDFでメモリ圧迫 | 高 | DPI制限・ページストリーミング |
| 座標変換のズレ | 高 | 単体テスト・デバッグ可視化 |

---

## 4. 依存関係

- PyMuPDF（PDFレンダリング）
- OpenCV（画像処理）
- FastAPI（API）
- pdfrw / PyPDF2（AcroForm 書き込み）
- TemporaryDirectory（セキュリティ）
