# research.md (010-docs-improvement)

## Decision: ドキュメント改善の技術的アプローチ
- README・設計ドキュメントの主要セクションを日本語・英語併記で記載
- 各ドキュメント末尾にGlossary（用語集）を設け、日本語・英語併記で用語を記載
- 主要な機能・手順の変更ごとに必ず該当ドキュメントを同時更新し、PRレビュー時にドキュメント差分も必須確認
- 各用語・モジュール名・ファイル名はプロジェクト内で重複禁止、英数字・ハイフンのみ使用
- 1プロジェクトあたり最大10万字程度、100ファイル未満を想定

## Rationale
- 多様な利用者（日本語・英語話者、非専門家含む）へのアクセシビリティ向上
- ドキュメントの品質・最新性を担保し、開発効率・保守性を高める
- プロジェクトの国際化・ナレッジ共有促進
- スケール・運用現実性を考慮

## Alternatives considered
- READMEのみ英語併記、設計ドキュメントは日本語のみ（国際化・一貫性に劣るため不採用）
- プロジェクトルートに共通glossary.md設置（分散管理・参照性の観点で却下）
- 定期レビューのみで最新性担保（運用形骸化リスク高いため不採用）
- 重要用語のみ注釈（網羅性・一貫性に劣るため不採用）
