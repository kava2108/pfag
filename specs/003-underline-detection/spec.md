# Feature Specification: 下線検出（テキストフィールド）

**Feature Branch**: `003-underline-detection`
**Created**: 2026-02-14
**Status**: Draft
**Input**: User description: "下線検出（テキストフィールド）機能：モルフォロジー演算による水平線強調、min_line_width・line_gap_thresholdパラメータ、バウンディングボックス化、フィールド配置ルール、単体テスト等（Issue #3）"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 明確な下線の検出 (Priority: P1)

ユーザーがアップロードした画像やPDFから、明確な下線（テキストフィールド用）を自動で検出できる。

**Why this priority**: 下線検出はフォーム自動化やデータ抽出の基盤となるため、最重要。

**Independent Test**: サンプル画像・PDFを用いて、下線が正しく検出されるかを確認。

**Acceptance Scenarios**:
1. **Given** 明確な下線が含まれる画像/PDF、**When** 検出処理を実行、**Then** 下線の位置・範囲が正しく出力される
2. **Given** 下線が複数存在する画像/PDF、**When** 検出処理を実行、**Then** すべての下線が検出される

---

### User Story 2 - パラメータ調整による柔軟な検出 (Priority: P2)

min_line_widthやline_gap_thresholdなどのパラメータを調整し、様々な下線パターンに対応できる。

**Why this priority**: 多様な帳票や手書きフォームに対応するため。

**Independent Test**: パラメータを変更し、異なる下線パターンで検出精度が変化することを確認。

**Acceptance Scenarios**:
1. **Given** 細い下線の画像、**When** min_line_widthを小さく設定、**Then** 細い下線も検出される
2. **Given** 途切れた下線の画像、**When** line_gap_thresholdを大きく設定、**Then** 連結して検出される

---

### User Story 3 - バウンディングボックス・フィールド配置ルール (Priority: P3)

検出した下線をバウンディングボックス化し、フィールド配置ルール（高さ・オフセット・インセット）に基づき出力できる。

**Why this priority**: 検出結果を後続処理やUIで活用するため。

**Independent Test**: バウンディングボックスの座標・サイズが仕様通り出力されることを確認。

**Acceptance Scenarios**:
1. **Given** 下線検出結果、**When** バウンディングボックス化処理を実行、**Then** 仕様通りの座標・サイズで出力される
2. **Given** 配置ルールが指定された場合、**When** 出力処理を実行、**Then** ルールに従った配置情報が得られる

---


### Edge Cases

- 画像内にノイズや不要な線が多い場合、誤検出を最小限に抑えられるか？
- 下線が極端に短い・途切れている場合、どこまで検出対象とするか？
- 下線が斜め・曲線の場合は検出対象外（水平線のみ検出）

## Clarifications
- Q: 非機能要件（パフォーマンス・スケーラビリティ・ログ・セキュリティ等）は？ → A: 1画像あたり処理時間1秒以内、1000x1000pxまで対応、ログはINFO以上、外部入力バリデーション必須
- Q: ノイズ・不要線の誤検出抑制の基準は？ → A: 最小幅30px・最小長さ50px・線の太さ2px以上で制御（デフォルト値）

- Q: バウンディングボックス・フィールド配置ルールの単位・デフォルト値は？ → A: 単位はpx、高さ20px・オフセット5px・インセット2px

## Requirements *(mandatory)*

### Functional Requirements
- **FR-008**: ノイズ・不要線の誤検出抑制のため、最小幅30px・最小長さ50px・線の太さ2px以上の閾値を設けること（デフォルト値）

- **FR-001**: システムは、画像またはPDFから明確な下線を自動検出できなければならない
- **FR-002**: モルフォロジー演算による水平線強調処理を実装すること
- **FR-003**: min_line_widthパラメータで検出する下線の最小幅を調整できること（デフォルト: 30px）
- **FR-004**: line_gap_thresholdパラメータで途切れ線の連結度合いを調整できること（デフォルト: 5px）
- **FR-005**: 検出した下線をバウンディングボックス化し、座標・サイズを出力できること
- **FR-006**: フィールド配置ルール（高さ・オフセット・インセット）を適用できること
- **FR-007**: 単体テストで明確な下線が正しく検出されることを確認する

### Key Entities

- **下線（Underline）**: 画像またはPDF内で検出された水平線。属性：開始座標、終了座標、幅、太さ
- **バウンディングボックス（BoundingBox）**: 下線を囲む矩形領域。属性：左上座標、幅、高さ（単位: px）
- **フィールド配置ルール（FieldLayoutRule）**: バウンディングボックスの高さ・オフセット・インセット等の設定値（単位: px、デフォルト: 高さ20px・オフセット5px・インセット2px）

## Success Criteria *(mandatory)*
## Non-Functional / Quality Attributes

- 1画像あたり処理時間1秒以内（1000x1000pxまで）
- ログ出力はINFOレベル以上
- 外部入力は必ずバリデーションを行うこと
- 画像サイズ・枚数のスケーラビリティは1000x1000px・1枚単位を想定

### Measurable Outcomes

- **SC-001**: サンプル画像・PDFで90%以上の下線が正しく検出される
- **SC-002**: パラメータ調整により、異なる下線パターンにも80%以上対応できる
- **SC-003**: バウンディングボックス・配置情報が仕様通り出力される
- **SC-004**: ユーザーからの検出精度に関する問い合わせが大幅に減少する
