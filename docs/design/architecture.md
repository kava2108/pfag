# PFAG アーキテクチャ設計書

## 1. 全体アーキテクチャ概要

PFAG（PDF Form Auto-Generator）は、静的PDFを入力とし、  
画像処理・座標変換・AcroForm 生成を経て、入力可能なPDFを出力するパイプライン型システムである。

主な構成要素は以下とする：

- PDF レンダリング層（PyMuPDF）
- 画像処理層（OpenCV）
- 座標変換層
- フォーム生成層（AcroForm）
- API 層（FastAPI）
- Agent 連携層

---

## 2. コンポーネント構成

### 2.1 PDF レンダリングコンポーネント

**責務：**

- 入力PDFをページ単位で画像化（300dpi）
- マルチページ対応
- 壊れたPDFの検出と例外化

**入出力：**

- 入力：PDF バイナリ
- 出力：ページごとの画像（メモリ or 一時ファイル）

---

### 2.2 画像処理コンポーネント（下線・矩形検出）

**責務：**

- 下線（テキストフィールド候補）の検出
- 矩形（チェックボックス候補）の検出
- パラメータ（感度・閾値）に基づくフィルタリング

**サブコンポーネント：**

- LineDetector
- CheckboxDetector

---

### 2.3 座標変換コンポーネント

**責務：**

- 画像座標（px）→ PDF座標（pt）の変換
- ページサイズ・解像度に応じたスケーリング
- 原点位置の差異（左上 vs 左下）の吸収

---

### 2.4 フォーム生成コンポーネント（AcroForm）

**責務：**

- `/Tx` `/Btn` フィールドの生成
- 既存 AcroForm への追記（append）
- 命名規則の適用（`field_p{page}_{type}{index}`）

---

### 2.5 API コンポーネント（FastAPI）

**責務：**

- `POST /v1/formify` のエンドポイント提供
- ファイルアップロード処理
- options JSON の解析
- ステータスコード・警告ヘッダの制御

---

### 2.6 Agent 連携コンポーネント

**責務：**

- PFAG API の呼び出しラッパ
- 感度調整ロジック（再試行）
- 警告メッセージのユーザー提示

---

## 3. データフロー

1. クライアント（または Agent）が PDF を `POST /v1/formify` に送信
2. PDF レンダリングコンポーネントがページ画像を生成
3. 画像処理コンポーネントが下線・矩形を検出
4. 座標変換コンポーネントが PDF座標に変換
5. フォーム生成コンポーネントが AcroForm フィールドを埋め込み
6. API コンポーネントが変換後PDFをレスポンスとして返却
7. Agent が結果をユーザーに提示し、必要に応じて再試行

---

## 4. 技術スタック

- 言語：Python 3.x
- PDF レンダリング：PyMuPDF
- 画像処理：OpenCV
- フォーム生成：pdfrw / PyPDF2
- API：FastAPI
- 実行環境：コンテナ（想定）

---

## 5. 非機能要件との対応

- パフォーマンス：ページ単位処理・DPI制御で対応
- セキュリティ：TemporaryDirectory・PDFサニタイズ
- 信頼性：検出0件時も 200 OK + 警告ヘッダで制御
